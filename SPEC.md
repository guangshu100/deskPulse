## 一、项目概述

### 1.1 产品定位

一款轻量级、跨平台的**久坐护眼提醒工具**，通过高情绪价值的提醒文案和护眼全屏页面，帮助用户养成健康的用眼习惯。

**AI增强版**：深度集成大语言模型，根据用户使用场景、时间、习惯生成**个性化、智能化**的提醒文案，让每次提醒都与众不同。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 智能监控 | 基于鼠标/键盘/触摸活动检测用户空闲状态 |
| 高情绪价值 | 20+条随机文案，温柔治愈风格，拒绝生硬命令 |
| **AI智能增强** | 大模型生成个性化提醒文案，上下文感知，智能学习用户偏好 |
| 护眼全屏 | 低饱和度背景色 + 护眼互动指引 + 极简交互 |
| 轻量便携 | Tauri打包 < 5MB，托盘静默运行，开机自启 |

### 1.3 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| **框架** | Tauri v2 | 比Electron更轻量（<5MB），Rust后端性能好 |
| **前端** | Vanilla JS + HTML/CSS | 无框架依赖，减少体积 |
| **存储** | tauri-plugin-store | 本地JSON存储，无数据库依赖 |
| **通知** | tauri-plugin-notification | 原生系统通知 |
| **AI集成** | OpenAI兼容API (硅基流动/本地部署) | 支持100+大模型，按需调用 |
| **打包** | tauri-bundler | 一键生成安装包 |

### 2.1 空闲状态检测

```rust
// Rust后端空闲检测逻辑
fn get_idle_time() -> u64 {
    // Windows: GetLastInputInfo
    // macOS: CGEventSourceSecondsSinceLastEventType
    // Linux: libxss (与Stretchly相同)

```

| 功能点 | 详细要求 | 默认值 | 可配置范围 |
|--------|----------|--------|------------|
| 检测方式 | 鼠标移动/点击、键盘输入、触摸操作 | - | - |
| 空闲判定 | 累计无操作时间达到阈值 | 30分钟 | 5-120分钟 |
| 检测间隔 | 定时轮询检测空闲时间 | 2秒 | 1-10秒 |
| 重置条件 | 检测到任意用户操作时立即重置 | - | - |

### 2.2 页面可见性联动

| 场景 | 行为 |
|------|------|
| 标签页隐藏/窗口最小化 | **暂停计时**，不触发提醒 |
| 标签页恢复/窗口恢复 | **恢复计时**，重新开始累计 |
| 切换到其他应用 | **保持计时**（用户仍在使用电脑） |

### 2.3 系统状态兼容

| 系统事件 | 行为 |
|----------|------|
| 系统熄屏/休眠 | **暂停计时** |
| 系统锁屏 | **暂停计时** |
| 系统唤醒/解锁 | **恢复计时**，弹出「回归提醒」弹窗（5秒后自动消失） |

## 三、提醒触发规则

### 3.1 触发流程图

```
用户空闲时间达到阈值
        │
        ▼
   弹出系统通知（非侵入）
        │
        ▼
   5秒内无操作？
   │         │
   │是       │否（用户点击/操作）
   ▼         ▼
全屏护眼页面   重置计时，继续监控
```

### 3.2 跳过机制

| 条件 | 行为 |
|------|------|
| 手动跳过 | 按 `Esc` 键或点击任意位置快速关闭 |
| 跳过次数限制 | 当天跳过超过3次 → 弹出「强制休息小贴士」弹窗 |
| 强制休息小贴士 | 显示一条额外的健康提醒文案，不强制阻断 |

### 3.3 参数配置

| 参数 | 默认值 | 可配置范围 | 说明 |
|------|--------|------------|------|
| `idleThreshold` | 30 | 5-120 | 空闲阈值（分钟） |
| `checkInterval` | 2 | 1-10 | 检测间隔（秒） |
| `notifyBeforeFullscreen` | 5 | 0-30 | 通知到全屏的延迟（秒） |
| `maxSkipsPerDay` | 3 | 1-10 | 每日允许跳过次数 |

## 四、高情绪价值内容设计

### 4.1 设计理念

高情绪价值的核心是让工具从「冷冰冰的提醒器」变成「贴心的健康伙伴」。从以下四个维度注入情绪价值：

| 维度 | 目标 | 实现方式 |
|------|------|----------|
| **个性化** | 让用户感觉「被懂」 | 身份标签、昵称、职业 |
| **互动性** | 让用户「愿意参与」 | 动态引导、成就反馈、随机彩蛋 |
| **情感共鸣** | 让用户「被关怀」 | 情绪感知、阶梯式提醒、温柔语气 |
| **场景适配** | 让用户「不被打扰」 | 时间感知、状态适配 |

### 4.2 文案库结构

```
通知文案库 (20+条)
│
├─ 温柔鼓励型 (5条)
├─ 幽默俏皮型 (5条)
├─ 暖心关怀型 (5条)
├─ 科普知识型 (5条)
└─ 随机抽取，避免重复
```

### 4.3 通知文案示例

```javascript
const notificationMessages = [

  "眼睛要放假啦！放下鼠标，喝口水，活动一下僵硬的肩膀呀～",
  "今日份休息提醒：久坐伤腰，站起来走两步，奖励自己一个深呼吸～",
```

### 13.8 API调用示例（Rust）

```rust
use serde::{Deserialize, Serialize};

#[derive(Debug, Serialize)]
struct ChatMessage {
    role: String,
    content: String,
}

#[derive(Debug, Serialize)]
struct ChatRequest {
    model: String,
    messages: Vec<ChatMessage>,
    temperature: f32,
    max_tokens: u32,
}

#[derive(Debug, Deserialize)]
struct ChatResponse {
    choices: Vec<Choice>,
}

#[derive(Debug, Deserialize)]
struct Choice {
    message: ChatMessage,
}

// 调用硅基流动API
fn generate_message(context: &Context) -> Result<String, Error> {
    let prompt = build_prompt(context);
    let messages = vec![
        ChatMessage { role: "system".to_string(), content: SYSTEM_PROMPT.to_string() },
        ChatMessage { role: "user".to_string(), content: prompt },
    ];
    
    let request = ChatRequest {
        model: config.model.clone(),
        messages,
        temperature: config.temperature,
        max_tokens: config.max_tokens,
    };
    
    let client = reqwest::Client::new();
    let response = client
        .post(&format!("{}/chat/completions", config.api_base_url))
        .header("Authorization", format!("Bearer {}", config.api_key))
        .header("Content-Type", "application/json")
        .json(&request)
        .send()?;
    
    let result: ChatResponse = response.json()?;
    Ok(result.choices[0].message.content)
}
```

### 15.8 「隐私友好」的个性化：无感知收集，不打扰

#### 15.8.1 「无感知」用户画像构建

不强制用户填写信息，通过使用行为自动生成标签：

| 行为特征 | 自动标记 | 用户可见性 |
|----------|----------|------------|
| 深夜使用 | user_tag: "熬夜党" | 可在设置中修改 |
| 频繁跳过 | user_status: "忙碌" | 可在设置中修改 |
| 按时休息 | user_status: "健康" | 可在设置中修改 |

#### 15.8.2 「本地优先」的 AI 处理

- 所有用户行为数据仅存储在本地，不上传服务器
- AI 调用时，上下文数据仅在本地组装
- 用户可随时清除所有数据
